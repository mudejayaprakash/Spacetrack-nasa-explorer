Logical Data Model

Our database architecture follows a relational model with clearly defined relationships between entities. The model demonstrates two one-to-many relationships essential for tracking space missions and observations.

Database Relationships Explained:

  • Projects to Observations (One-to-Many - 1:N): Each project serves as a container for multiple related observations. One project can have many observations, but each observation belongs to exactly one project. This relationship is established through the project_id foreign key in the Observations table.
  • Dynamic Data Management: When you create, update, or delete observations through the CRUD interface, the dashboard automatically reflects these changes in real-time. All charts, statistics, and the activity heatmap are dynamically generated from your actual database records.
  • Data Integrity: The relational structure ensures that observations are properly linked to their parent projects, maintaining data consistency and enabling powerful filtering capabilities across the application.
