Logical Data Model

Our database architecture follows a relational model with clearly defined relationships between entities. The model demonstrates two one-to-many relationships essential for tracking space missions and observations.

Database Relationships Explained:

  • Missions to Observations (One-to-Many): A single mission can generate multiple observations over time. For example, the Perseverance Mars mission has created hundreds of observations including images, rock samples, and atmospheric readings.
  • Celestial Objects to Observations (Many-to-Many): A single celestial object can be observed multiple times by different missions or the same mission at different times. For instance, Mars has been observed by Hubble, multiple Mars rovers, and orbital missions.

How This Fits Our Application:

This relational structure enables SpaceTrack to efficiently track which missions observed which celestial objects and when. Users can query observations by mission, by celestial object, or by date range. The CRUD operations in our application directly interact with these tables, allowing users to create new observations, view existing records, update observation details, and remove outdated entries.
